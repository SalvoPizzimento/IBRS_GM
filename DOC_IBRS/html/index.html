Data Sharing Infrastructure IBRS
Progetto finale per Sistemi Cloud (ideato da Alessandro Midolo e Salvatore Pizzimento)

Lo scopo del progetto è quello di creare un'infrastruttura cloud-based per fornire un servizio di data sharing basato sullo schema crittografico IBRS (Identity Based Ring Signature).
La struttura di base è composta da 3 entità:

  • Group Admin
  • Group Member
  • Cloud Server

Il Group Admin sarà il responsabile della creazione e condivisione dei parametri e delle chiavi per i membri di un gruppo di condivisione.
Il Group Member sarà l'entità che vorrà creare un gruppo di condivisione o farne parte comunicando con il Group Admin; comunicando con il Cloud Server potrà effettuare download e/o upload di file sullo spazio di storage fornito da Amazon Web Services S3.
Il Cloud Server sarà il principale responsabile delle operazioni di scambio di contenuti tra i membri legittimi di uno stesso gruppo di condivisione.

Specifiche tecniche

Sia il Group Admin che il Cloud Server risiederanno su istanze Amazon Web Services EC2, con Security Groups customizzati per permettere dall'esterno i comandi ping, ssh, e per accettare connessioni TCP nel range di porte 8000-9000.
Il core del Group Member sarà un'immagine Docker opportunamente creata per una facile esecuzione.
Tutte le entità richiedono le librerie crittografiche PBC, Nettle, GMP per implementare lo schema di firma IBRS, il compilatore gcc e altre utilities di sistema.

Installazione

Per installare il server Group Admin è necessario creare una nuova istanza EC2 con immagine Ubuntu. I Security Group customizzati permettono il comando di ping dall'esterno, l'accesso ssh da host esterni e la corretta creazione di connessioni TCP per la comunicazione socket tra i due server e i client. Dopo l'accesso al server è necessario scaricare le librerie crittografiche e le utilities di sistema per avviare il progetto correttamente. È necessario settare opportunamente l'indirizzo ip del Cloud Server nella variable d'ambiente "CS" tramite comando 'export CS=IP'. Successivamente bisogna scaricare il core del Group Admin tramite comando 'git' e avviare il building tramite comando 'make'. Concluso il building è possibile mettere in ascolto il server avviando il progetto con il comando './test-ibrs-gm'.

Per installare il server Cloud Server è necessario creare una nuova istanza EC2 con immagine Ubuntu. I Security Group customizzati permettono il comando di ping dall'esterno, l'accesso ssh da host esterni e la corretta creazione di connessioni TCP per la comunicazione socket tra i due server e i client. Dopo l'accesso al server è necessario scaricare le librerie crittografiche e le utilities di sistema per avviare il progetto correttamente. È necessario configurare opportunamente l'accesso al bucket S3, tramite i comandi della CLI di AWS, che servirà da storage per il server. Successivamente bisogna scaricare il core del Cloud Server tramite comando 'git' e avviare il building tramite comando 'make'. Concluso il building è possibile mettere in ascolto il server avviando il progetto con il comando './test-ibrs-cs'.

Per installare il client Group Member è necessario buildare l'immagine Docker a partire dal Dockerfile fornito. Concluso tale processo, sarà necessario creare un nuovo container e settare opportunamente gli indirizzi IP del Group Admin e del Cloud Server attraverso i comandi 'export GA=IP' e 'export CS=IP'. Successivamente bisogna avviare il building tramite comando 'make'. Concluso il building è possibile mettere avviare il client del progetto con il comando './test-ibrs-gm'.

Utilizzo

L'utente potrà scegliere se:

  • collegarsi al Group Admin per creare o partecipare ad un gruppo di condivisione, ricevendo così i parametri necessari per l'algoritmo e la coppia di chiavi per l'autenticazione;
  • collegarsi al Cloud Server per effetturare download o upload di contenuti nel gruppo di condivisione d'appartenenza, fornendo come prova d'autenticazione la firma del nome del file richiesto.
